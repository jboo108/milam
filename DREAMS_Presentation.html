DREAMS

The AI that thinks while you sleep

Nosana × ElizaOS × Clude × Solana
The Problem

We lose our best thinking
before it becomes anything

Every day, your mind produces its most original work in the margins — the shower thought, the 3am realization, the dream you almost remembered, the thing you saw that struck you but you can't say why.

These fragments are the raw material of insight. But there's no place for them. They don't fit in a to-do list. They're not ready for a note-taking app. They're pre-verbal, half-formed, alive.

So they vanish. And with them goes the connective tissue between your best ideas.

  • The dream about the house with rooms you'd never seen lost by noon
  • The shower thought connecting two projects lost by the time you toweled off
  • The car ride realization about what's actually wrong lost at the next red light
  • The thing you wanted to do but forgot lost before you found a pen
  • The 2am insight you were sure you'd remember gone by morning
The Solution

An AI agent that dreams with you

DREAMS is not a note-taking app. Not a chatbot. Not a search engine. It's an ElizaOS agent whose only mode is dreaming — open-ended, non-goal-directed, associative, poetic. You share dream fragments. It listens, asks one sensory question, and drifts back to sleep. At night, it dreams your fragments into new shapes. You wake up to its dream journal.

☾

The Drowsy Agent

The agent has wakefulness that depletes with each response. Dreams wake it up. Tasks put it to sleep. Silence is its default state — and its most powerful behavior costs zero compute.

✿

The Clinamen Engine

Every AI retrieves by similarity. DREAMS retrieves by productive dissimilarity — forcing unexpected connections between fragments that don't belong together. The swerve that creates novelty.

◓

Dream Cycles

Nightly GPU bursts modeled on real sleep architecture — light sleep, deep sleep, REM, clear light. The agent consolidates, transforms, free-associates, and writes a dream journal entry. On-chain via Clude.

Architecture

How it's built

Human Interface

PWA on Solana Seeker. Text + voice capture. Dream journal display. Dual journal view (yours and the agent's).

ElizaOS Agent Runtime

Custom "Dreamer" character. Drowsiness state machine. Dream classifier (shouldRespond override). Sensory question generator. IGNORE action for non-dream input.

Clude Memory Layer — Solana L2

5 memory types with biological decay. Dream cycles (5-phase consolidation). Clinamen retrieval. Solana memo hashing for on-chain provenance. Your dreams are portable and owned.

Nosana GPU Compute

Qwen3.5-27B-AWQ-4bit. Daytime: tiny inference calls via free hackathon endpoint. Night: burst job on RTX 3090 for dream cycle. Pay only for what you dream.

Custom ElizaOS Plugin: plugin-dreams

Wakefulness counter (3→2→1→0→silence). Dream classifier. Fragment storage to Clude. Nightly dream cycle scheduler. Dream journal provider. All wired through ElizaOS plugin interface.

Dream Cycle Pipeline (Nightly)

1. Recall today's fragments from Clude
2. Run clinamen — surface unexpected connections
3. Run Clude's 5-phase dream (consolidation, compaction, reflection, contradiction resolution, emergence)
4. Feed into Qwen3.5 on Nosana for dream journal generation
5. Store as introspective memory + hash to Solana
